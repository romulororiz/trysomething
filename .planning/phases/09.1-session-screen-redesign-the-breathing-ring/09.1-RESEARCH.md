# Phase 9.1: Session Screen Redesign — The Breathing Ring - Research

**Researched:** 2026-03-22
**Domain:** Flutter CustomPainter, canvas-based progress ring, animation, film grain, immersive UI
**Confidence:** HIGH

## Summary

This phase replaces the existing 103-particle field system (`SessionParticleField` + `_ParticleField3DPainter` at 644 lines) with a premium Apple Watch-style progress ring that serves as the single visual anchor across all 4 session phases. The ring is drawn via `Canvas.drawArc` with a layered approach: track circle, progress arc, glow arc (via `MaskFilter.blur`), and a leading dot. The breathing animation uses a sinusoidal `AnimationController` driving `Transform.scale` on the ring widget.

The existing codebase already demonstrates all the required Canvas techniques: `drawArc` in `RadialHoldPainter`, `MaskFilter.blur` glow layers in `BrushstrokeTimerPainter`, noise-based organic movement in `particle_timer_painter.dart`, and the `_RollingTimer`/`_RollingDigit`/`_Separator` components which are explicitly preserved. The session state machine (`SessionNotifier` + `SessionState`) is solid and untouched. The main work is replacing visual layers in the session screen, not changing data flow.

Film grain is best implemented as a static pre-rendered PNG overlay at 1-2% opacity (single `Image.asset` with `BlendMode`), avoiding per-frame CustomPainter noise which would be needlessly expensive. The breathing background (D-09) is a simple `AnimatedContainer` or `TweenAnimationBuilder` cycling between `#0A0A0F` and `#0F0F14`.

**Primary recommendation:** Create a single `BreathingRingPainter` CustomPainter that draws all 4 layers (track, progress arc, glow, leading dot) and is driven by `progress`, `breathScale`, and `glowIntensity` parameters. Wrap it in a `BreathingRing` StatefulWidget that owns the breathing `AnimationController`. Place this ring in the session screen's `Stack` as the visual constant, with phase-specific content crossfading around it via `AnimatedSwitcher`.

### Pattern 1: Layered Ring Painter
**What:** A single `CustomPainter` that draws the ring in 4 layers (bottom to top): track circle, glow halo, progress arc, leading dot. This avoids multiple `CustomPaint` widgets stacked and reduces draw call overhead.
**When to use:** The timer phase and any phase showing the ring.
**Example:**
```dart
class BreathingRingPainter extends CustomPainter {
  final double progress;      // 0.0 to 1.0
  final double breathScale;   // ~1.0 to 1.008
  final double glowIntensity; // 0.15 normal, 0.25 last minute
  final double ringOpacity;   // 1.0 normal, 0.4 paused

  BreathingRingPainter({
    required this.progress,
    this.breathScale = 1.0,
    this.glowIntensity = 0.15,
    this.ringOpacity = 1.0,
  });

  @override
  void paint(Canvas canvas, Size size) {
    final center = Offset(size.width / 2, size.height / 2);
    final radius = size.width / 2; // Widget is already sized to 260-280

    // Apply breathing scale
    canvas.save();
    canvas.translate(center.dx, center.dy);
    canvas.scale(breathScale, breathScale);
    canvas.translate(-center.dx, -center.dy);

    final rect = Rect.fromCircle(center: center, radius: radius);
    const startAngle = -pi / 2; // 12 o'clock
    final sweepAngle = 2 * pi * progress;

    // Layer 1: Track ring (D-05)
    canvas.drawCircle(center, radius, Paint()
      ..color = const Color(0x0FFFFFFF) // 6% white
      ..style = PaintingStyle.stroke
      ..strokeWidth = 4.5);

    // Layer 2: Glow halo (D-01)
    if (progress > 0) {
      canvas.drawArc(rect, startAngle, sweepAngle, false, Paint()
        ..color = AppColors.accent.withValues(alpha: glowIntensity * ringOpacity)
        ..style = PaintingStyle.stroke
        ..strokeWidth = 8.0
        ..strokeCap = StrokeCap.round
        ..maskFilter = const MaskFilter.blur(BlurStyle.normal, 8.0));
    }

    // Layer 3: Progress arc (D-01, D-02)
    if (progress > 0) {
      canvas.drawArc(rect, startAngle, sweepAngle, false, Paint()
        ..color = AppColors.accent.withValues(alpha: ringOpacity)
        ..style = PaintingStyle.stroke
        ..strokeWidth = 4.5
        ..strokeCap = StrokeCap.round);
    }

    // Layer 4: Leading dot (D-04)
    if (progress > 0 && progress < 1.0) {
      final dotAngle = startAngle + sweepAngle;
      final dotX = center.dx + radius * cos(dotAngle);
      final dotY = center.dy + radius * sin(dotAngle);
      // Soft glow behind dot
      canvas.drawCircle(Offset(dotX, dotY), 8, Paint()
        ..color = AppColors.accent.withValues(alpha: 0.15 * ringOpacity)
        ..maskFilter = const MaskFilter.blur(BlurStyle.normal, 4));
      // Solid dot
      canvas.drawCircle(Offset(dotX, dotY), 3, Paint()
        ..color = AppColors.accent.withValues(alpha: 0.8 * ringOpacity));
    }

    canvas.restore();
  }

  @override
  bool shouldRepaint(BreathingRingPainter old) =>
      old.progress != progress ||
      old.breathScale != breathScale ||
      old.glowIntensity != glowIntensity ||
      old.ringOpacity != ringOpacity;
}
```

### Pattern 2: Breathing Animation Controller
**What:** A dedicated `AnimationController` in the ring widget that cycles sinusoidally. The cycle duration changes based on session state (active/paused/last-minute).
**When to use:** Always active when ring is visible.
**Example:**
```dart
// In BreathingRing StatefulWidget:
late AnimationController _breathController;
double _breathScale = 1.0;

void _updateBreathCycle(Duration duration) {
  _breathController.duration = duration;
  if (!_breathController.isAnimating) {
    _breathController.repeat();
  }
}

// In listener:
_breathScale = 1.0 + 0.008 * sin(_breathController.value * 2 * pi);
// D-10: Active = 4s cycle, max 1.008
// D-11: Paused = 8s cycle
// D-12: Last minute = 3s cycle
```

### Pattern 3: Ring as Visual Constant Across Phases
**What:** The ring widget lives in `session_screen.dart` Stack (NOT inside individual phase widgets). Phase content crossfades around it. The ring receives `progress` from `_computeProgress(session)` which already returns 0.0 for prepare, fills during timer, and 1.0 for completing/reflect/complete.
**When to use:** The ring MUST be in the session_screen.dart layer, not per-phase.
**Why:** This prevents the ring from being destroyed/recreated on phase transitions, enabling smooth morphing animations. The existing `_computeProgress` method already handles this correctly.

### Pattern 4: Session Screen Layer Stack
**What:** Replace the current 2-layer Stack (particle field + phase content) with a 5-layer Stack:
```
Layer 1: Breathing background (AnimatedContainer pulsing #0A0A0F to #0F0F14)
Layer 2: Ambient gradient spotlight (RadialGradient, coral at 3-5%)
Layer 3: Film grain overlay (static PNG at 1-2% opacity)
Layer 4: Breathing ring (BreathingRing widget, positioned at ~42% from top)
Layer 5: Phase content (AnimatedSwitcher with fade+slide)
```

### Anti-Patterns to Avoid
- **Per-frame noise generation:** Do NOT use `CustomPainter` with `Random()` to generate film grain per frame. A static PNG is visually identical and costs zero CPU.
- **Multiple CustomPaint widgets for ring layers:** Do NOT draw track, arc, glow, and dot as 4 separate `CustomPaint` widgets. One painter with 4 `canvas.draw*` calls is far more efficient.
- **Ring inside phase widgets:** Do NOT put the ring inside `SessionTimerPhase` or other phase widgets. It must live in `session_screen.dart` to survive phase transitions without flickering.
- **Rebuilding AnimationController on phase change:** Do NOT recreate the breathing controller when switching phases. Change its `duration` property instead.
- **Using `BackdropFilter` for ring glow:** `MaskFilter.blur` on `Paint` is far cheaper than `BackdropFilter` which blurs everything behind it. The ring glow should use `MaskFilter.blur`.

## Common Pitfalls

### Pitfall 1: shouldRepaint Always Returning True
**What goes wrong:** The existing `_ParticleField3DPainter` returns `shouldRepaint => true`, causing 60fps repaints even when nothing changes. This is fine for particles that move every frame, but wrong for a ring that only changes when progress ticks (once per second).
**Why it happens:** Copy-paste from particle code.
**How to avoid:** Implement proper `shouldRepaint` comparing `progress`, `breathScale`, `glowIntensity`, `ringOpacity`. The breathing animation WILL change every frame, so the controller listener should call `setState` which triggers repaint naturally.
**Warning signs:** Battery drain reports, janky scrolling if ring widget somehow gets into a scrollable.

### Pitfall 2: MaskFilter.blur Performance on Older Devices
**What goes wrong:** `MaskFilter.blur` on a `Paint` used with `drawArc` creates a software-rasterized blur per frame. On very old GPUs this can drop frames.
**Why it happens:** The blur sigma is applied per-pixel on the arc path.
**How to avoid:** Use `RepaintBoundary` around the ring widget. Keep blur sigma reasonable (8.0 for glow is fine). The ring is only ~280dp -- this is a small area to blur, not full-screen. Test on Nothing Phone 3a specifically.
**Warning signs:** Frame time spikes in Flutter DevTools performance overlay during timer phase.

### Pitfall 3: AnimationController Duration Change Mid-Animation
**What goes wrong:** Calling `_breathController.duration = newDuration` while the controller is animating does not smoothly transition. The controller jumps or resets.
**Why it happens:** Flutter's `AnimationController.duration` setter does not interpolate.
**How to avoid:** When switching breath cycle (active -> paused -> last minute), call `_breathController.stop()`, set new duration, then `_breathController.repeat()`. Or use a manual `Timer.periodic` approach with `sin(time * frequency)` instead of relying on controller duration. The particle field code already uses this pattern (`_time += dt; sin(_time * frequency)`).
**Warning signs:** Breathing animation "jumping" when pausing or entering last minute.

### Pitfall 4: Timer Digits Inside Ring Sizing
**What goes wrong:** IBM Plex Mono 72pt digits (D-22) at 4 characters + separator may not fit inside a 260-280dp ring diameter. The inner diameter available for text is `260 - 2*4.5(stroke) - 2*8(glow) = ~235dp`. Font metrics for 72pt IBM Plex Mono: each digit ~44px wide, total "00:00" ~200px wide + ~20px separator = ~220px. Height ~80px. This fits, but barely.
**Why it happens:** Ring diameter and font size were chosen independently.
**How to avoid:** Use the existing `_RollingTimer` component but update its font size to 72pt (currently 64pt). Verify visually that digits fit with comfortable margins. Keep `_RollingDigit` width at ~44-46dp and height at ~80dp. If too tight, reduce to 64pt (current) -- D-22 says "72pt" but the rolling animation mechanics need clearance.
**Warning signs:** Digits clipped by ring border, or digits overlapping the ring arc.

### Pitfall 5: Coach Tip Not Passed to Session Screen
**What goes wrong:** `coachTip` field exists on `RoadmapStep` but is NOT currently passed from the home screen navigation to the session screen. The `extra` map in all 3 launch sites in `home_screen.dart` does not include `coachTip`.
**Why it happens:** D-19 (coach tip on-demand) is a new feature -- the session screen never needed it before.
**How to avoid:** Add `coachTip` as an optional `String?` parameter to `SessionScreen`, add it to the GoRouter `extra` map in `router.dart`, pass `step.coachTip` from all 3 launch sites in `home_screen.dart`, and store it in `SessionState`.
**Warning signs:** Coach tip icon shows but bottom sheet is always empty or hidden.

### Pitfall 6: Test File Has Stale Constructor
**What goes wrong:** The existing `session_timer_test.dart` creates `SessionScreen(...)` and `_NullSessionNotifier` without the `completionMessage` parameter. Adding `coachTip` will also require updating the test.
**Why it happens:** Test was written before `completionMessage` was added (the `_NullSessionNotifier.startSession` signature is missing it).
**How to avoid:** Update the test file's `_pumpScreen` helper and `_NullSessionNotifier.startSession` override to include both `completionMessage` and `coachTip` parameters.
**Warning signs:** Tests fail to compile after adding `coachTip` parameter.
