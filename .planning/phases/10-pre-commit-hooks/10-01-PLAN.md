<task type="auto">
  <name>Task 1: Install Lefthook and create hook configuration</name>
  <files>server/package.json, lefthook.yml</files>
  <read_first>
    - server/package.json (current devDependencies)
    - server/tsconfig.json (confirm tsc configuration)
    - analysis_options.yaml (confirm dart analyzer rules)
    - .gitignore (check nothing blocks lefthook)
  </read_first>
  <action>
1. Install Lefthook as a devDependency in server/ (per D-01, using npm in server/ per D-02 discretion):
   ```bash
   cd server && npm install --save-dev lefthook
   ```

2. Create `lefthook.yml` at the repository root (per D-05) with this exact content:
   ```yaml
   pre-commit:
     parallel: true
     commands:
       flutter-analyze:
         glob: "*.dart"
         run: dart analyze lib/
       typescript-typecheck:
         glob: "server/**/*.ts"
         root: "server/"
         run: npx tsc --noEmit
   ```

   Key configuration choices (per D-03):
   - `parallel: true` — both checks run simultaneously for speed (per D-04, must complete within 30 seconds)
   - `flutter-analyze` runs `dart analyze lib/` on all lib/ files when any *.dart file is staged. Using `dart analyze lib/` (not `flutter analyze`) because it is faster and sufficient for analysis errors. The glob acts as a gate — the job only runs when Dart files are staged.
   - `typescript-typecheck` runs `npx tsc --noEmit` from the server/ directory when any *.ts file in server/ is staged. Uses the existing `tsconfig.json` settings. Uses `npx tsc --noEmit` directly (not `npm run lint`) to avoid extra npm overhead for hook speed.
   - No dart-format check (not in requirements D-03)
   - No freezed-check warning (not in requirements D-03)
   - No commit-msg hook (not in requirements, keeping scope tight)

3. Initialize Lefthook hooks in the git repository:
   ```bash
   npx --prefix server lefthook install
   ```
   This creates the actual git hook scripts in `.git/hooks/` that delegate to Lefthook.

4. Verify `lefthook.yml` is NOT in `.gitignore` (it should be committed per D-05). The current `.gitignore` does not exclude it, so no changes needed.
  </action>
  <verify>
    <automated>test -f lefthook.yml && grep -q "pre-commit" lefthook.yml && grep -q "flutter-analyze" lefthook.yml && grep -q "typescript-typecheck" lefthook.yml && echo "PASS: lefthook.yml exists with both hook jobs" || echo "FAIL"</automated>
  </verify>
